UserALE API¶

class userale.ale.Ale(output='userale.log', autostart=True, interval=5000, user=None, version=None, details=False, resolution=500, shutoff=[])¶

Bases: PyQt5.QtCore.QObject

UserALE intercepts all application events by letting the developer install an event filter in their PyQT5 application to generate detailed user logs. UserAle does not capture system level logs or events generated by a non-user (a.k.a. system and signals sent between QObjects).

Parameters:
  • output – [str] The file or url path to which logs will be sent
  • autostart – [bool] Should UserAle start auotmatically on app rendering
  • interval – [int] The minimum time interval in ms betweeen batch transmission of logs
  • user – [str] Identifier for the user of the application
  • version – [str] The application version
  • details – [bool] Should detailed logs (key strokes, input/change values) be recorded
  • resolution – [int] Delay in ms between instances of high frequency logs like mouseovers, scrolls, etc
  • shutoff – [list] Turn off logging for specific events. For example, to ignore mousedown events, [‘mousedown’]

An example log will appear like this:

{
    'target': ,
    'path': ['Example', 'testLineEdit'],
    'clientTime': ,
    'location': {'x': 82, 'y': 0},
    'type': 'dragstart',
    'userAction': 'true',
    'details' : [ ],
    'userId': 'userABC1234',
    'toolVersion': 'myApplication',
    'useraleVersion': '1.0.0 alpha'
}
